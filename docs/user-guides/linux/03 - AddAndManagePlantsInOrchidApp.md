# OrchidApp

# Add and Manage Plants in OrchidApp

User guide for creating plant records and using the Plant Details page

**Applies to:** Windows 11

Use this guide after OrchidApp has been installed or upgraded using the Windows installer, and the first-use setup has been completed.

---

# Purpose

This guide explains how to:

- add a plant
- understand the Plant Details page
- record day-to-day events
- add and manage photos
- record flowering
- record repotting
- move a plant between growing locations
- split a plant
- propagate a plant
- view plant heritage

---

# Before you start

Before adding a plant:

- add the plant group, also called the genus
- add the species / hybrid before adding the plant
- add the growing locations you use
- add the growth media you use

Each plant gets a permanent **Plant Tag** automatically when it is saved.

Use events to build the plant history over time rather than keeping everything in one large note.

---

# Typical workflow

1. Open **Plants** from the Home page or the top menu.
2. Choose **Add plant**.
3. Select the species / hybrid.
4. Enter acquisition details and save the plant.
5. Use the Plant Details page to add observations, photos, movements, flowering and repotting records.

---

# Quick reference: Plant Details actions

| Action | Use it for | Result |
|---|---|---|
| Add observation | General notes or quick observations, including photo observations. | Adds an observation card to the plant history. |
| Move plant | Recording that the plant has moved to a new growing location. | Updates the current location and keeps previous locations in history. |
| Add flowering | Recording a flowering period, spike count, flower count and notes. | Adds flowering to history and updates the summary on the Plant Details page. |
| Add repotting | Recording potting medium, root condition, pot size and reason for repotting. | Adds repotting to history and updates the repotting summary. |
| Split plant | Ending one plant and creating two or more new independent plants. | The parent becomes ended/read-only. Child plants receive new permanent Plant Tags. |
| Propagate plant | Creating a new plant while keeping the existing plant alive. | Creates a new plant linked to the original plant as provenance. |
| Tap image | Opening the plant photo page. | Lets you browse photos, upload more and set the main image. |

# 2. Add a plant

## Select the species / hybrid

OrchidApp first asks which species / hybrid the new plant belongs to.

This is why the species / hybrid should already exist before adding a plant.

**Suggested screenshot:** Choose the botanical name for the plant.

## Enter plant details

The plant name is optional.

The acquisition date and acquisition source are useful for collection history, but they can be left blank if unknown.

**Suggested screenshot:** Enter optional plant details before saving.

# 3. Save the plant and note the Plant Tag

Use the date picker for the acquisition date, if known.

When you save, OrchidApp automatically creates a permanent **Plant Tag**.

The Plant Tag is the plant’s stable ID. It should not be changed or reused.

It is recommended that the physical plant label includes the Plant Tag.

**Suggested screenshot:** Use the date picker for the acquisition date, if known.

**Suggested screenshot:** After saving, OrchidApp displays the new permanent Plant Tag.

# 4. Understand the Plant Details page

The Plant Details page is the main working page for each plant.

It shows:

- the permanent Plant Tag
- current location
- acquisition details
- recent repotting information
- recent flowering information
- plant history
- available plant actions

**Suggested screenshot:** The Plant Details page showing the current plant summary and history actions.

The Plant Details page works as follows:

- The **Edit** button changes basic plant details for corrections.
- The large action buttons add new history records.
- History cards show observations, moves, flowering and repotting in date order.
- The plant image opens the Photos page when tapped.

---

# 5. Add an observation

Use observations for ordinary notes about the plant, such as:

- leaf condition
- root growth
- pests
- treatment notes
- watering notes
- general care comments

To add an observation:

1. Open the plant.
2. Choose **Add observation**.
3. Enter the observation date.
4. Type the notes.
5. Choose **Add observation**.

**Suggested screenshot:** Add a general observation.

The observation appears in the plant history.

---

# 6. Add a photo observation

Photos are added through observations.

Use **Add photo** when the main purpose of the observation is to attach one or more plant photos.

To add a photo observation:

1. Open the plant.
2. Choose **Add observation**.
3. Choose **Add photo**.
4. Choose **Browse**.
5. Select the image file or files.
6. Save the observation.

The photo observation is added to the history.

**Suggested screenshot:** Choose Add photo on the observation screen.

**Suggested screenshot:** Select the photo file and choose Open.

# 9. Move a plant

Use **Move plant** when the plant changes growing location.

This updates the current location and preserves previous locations in the plant history.

To move a plant:

1. Choose **Move plant** on the Plant Details page.
2. Select the new location.
3. Enter the move date.
4. Add a move reason or location notes, if useful.
5. Choose **Move plant**.

**Suggested screenshot:** Record a plant move to a new location.

The current location is shown in the plant summary.

# 10. Add flowering

Use **Add flowering** to record when a plant flowers.

Leave the end date blank if the plant is still flowering.

Flowering fields include:

- **Start date** — when flowering began
- **End date** — optional, and can be added later
- **Spike count** — useful for comparing future flowering
- **Flower count** — useful for comparing future flowering
- **Notes** — anything useful about the flowering

**Suggested screenshot:** Record a flowering period.

Flowering appears in the history and summary.

---

# 11. Edit flowering when it ends

If you left the end date blank, use **Edit event** on the flowering history card when flowering has finished.

Add the end date and save.

**Suggested screenshot:** Add the flowering end date when the flowering period ends.

The flowering history updates after editing.

---

# 12. Add repotting

Use **Add repotting** to record potting changes, medium condition and the reason for repotting.

This helps build a useful care history.

Repotting fields include:

- **Previous growing medium** — what the plant was in before repotting
- **Condition notes** — root, pot or medium condition
- **New growing medium** — what the plant was repotted into
- **Reason for repotting** — why the change was made

**Suggested screenshot:** Record repotting details and save.

The repotting record appears in the history and updates the summary.

---

# 13. Edit or remove history events

Each history card has **Edit event** and **Remove event** buttons where the event type allows changes.

Use these to correct mistakes rather than creating duplicate records.

Use **Remove event** carefully.

For ordinary corrections, editing the existing event is usually better than removing it and adding a new one.

---

# 14. Split or propagate a plant

Split and propagation are more advanced than ordinary history events.

They both create new plant records, but they mean different things.

## Split plant

Use **Split plant** when the original plant is ending and becoming two or more new independent plants.

The original plant becomes ended/read-only.

The new plants receive their own permanent Plant Tags and independent histories.

## Propagate plant

Use **Propagate plant** when creating a new plant while keeping the original plant active.

Use it for propagation such as:

- keiki
- backbulb
- cutting
- similar propagation methods

The new plant is linked to the original plant as provenance.

**Suggested screenshot:** Use Propagate plant when the original plant remains active.

**Suggested screenshot:** Use Split plant when the original plant is ending and becoming new plants.

---

# 15. Split a plant

Splitting is permanent.

OrchidApp warns that the original plant will end and new plants will be created.

Enter:

- split date
- reason
- notes
- details for each new plant

Confirm the split only when you are sure.

**Suggested screenshot:** Split plant confirmation and new plant details.

---

# 16. After a split

After the split, OrchidApp shows the new permanent IDs.

The original plant becomes ended and read-only.

The new child plants are linked back to the parent through heritage.

**Suggested screenshot:** The split creates new plant IDs.

**Suggested screenshot:** The ended parent remains visible and read-only, with child plants shown in Heritage.

---

# 17. View plant heritage

When a plant has a parent or child relationship, the **Heritage** section appears on the Plant Details page.

Use it to move between related plants and view the full lineage.

A child plant has its own Plant Tag and its own independent history.

**Suggested screenshot:** A child plant shows its parent in the Heritage section.

# Plant management checklist

- Create the genus and species / hybrid before adding the plant.
- Use **Add plant** for each individual plant in the collection.
- Record ordinary care notes as observations.
- Add photos through observations or the Photos page.
- Use **Move plant** for location changes.
- Use **Add flowering** and **Add repotting** for structured care history.
- Use **Split plant** only when the original plant ends.
- Use **Propagate plant** when the original plant stays active.
- Use the Plant Tag to find and identify the plant later.

---

# Common mistakes to avoid

- Do not create a new plant just because it moved location.
- Do not create a new plant just because it was repotted.
- Do not split a plant unless the original plant is ending.
- Do not rely on memory for plant identity: use the Plant Tag.
